# 轻瘦 - 减肥计划 App 产品需求文档 (PRD)

## 1. 产品概述

**一句话说明**：一款帮助用户统一管理减肥计划、饮食记录、体重变化和轻断食的移动端应用，解决减肥数据散落在各平台无法集中管理的痛点。

**给谁用**：有减肥需求、使用轻断食方法、需要记录饮食和体重的个人用户。

**解决什么问题**：
- 减肥相关数据分散在小程序、Excel、便签等多个地方，无法统一查看
- 轻断食时间靠记忆，容易忘记开始/结束时间
- 饮食热量计算麻烦，不知道每天吃了多少
- 体重变化没有可视化趋势，难以坚持

## 2. 用户角色

### 主要使用者画像

| 维度 | 描述 |
|------|------|
| **身份** | 25-40岁的都市白领/上班族 |
| **性别** | 男女都有，女性偏多 |
| **减肥方式** | 尝试或正在使用轻断食（16:8等） |
| **痛点** | 试过多种减肥方法难以坚持，数据分散管理麻烦 |
| **使用场景** | 饭前记录饮食、早上起床称体重、工作间隙看断食进度 |
| **技术水平** | 普通智能手机用户，能熟练使用常用App |

## 4. 页面结构

### 4.1 底部导航 Tab（5个）

```
[首页]  [饮食]  [断食]  [体重]  [我的]
```

### 4.2 页面详细设计

#### 页面1：首页（Dashboard）

**核心模块**：
1. **顶部问候区**：用户头像 + 昵称 + 今日日期 + BMI
2. **断食状态卡片**：
   - 显示"断食中 / 进食中"状态标签
   - 开始时间 / 结束时间
   - 倒计时（圆形进度条）
   - 快捷按钮：开始断食 / 结束断食
3. **体重进度卡片**：
   - 当前体重 vs 目标体重
   - 进度条（百分比）
   - 已减重 / 还需减重
   - 快捷按钮：记录体重
4. **今日热量卡片**：
   - 已摄入 / 推荐摄入
   - 环形进度图
   - 三大营养素比例（蛋白/碳水/脂肪）
   - 快捷按钮：记录饮食
5. **今日打卡日历入口**：显示连续打卡天数

**关键交互**：
- 点击快捷按钮弹出对应操作弹窗（快速记录，不用跳转页面）
- 下拉刷新最新数据
- 左右滑动卡片可切换不同统计维度

---

#### 页面2：饮食（Food）

**核心模块**：
1. **顶部搜索栏**：搜索框 + 扫码（P2）
2. **今日摄入概览条**：已摄入热量 / 剩余热量
3. **餐次分类Tab**：早餐 / 午餐 / 晚餐 / 加餐
4. **分类入口网格**：主食 / 肉类 / 蔬菜 / 水果 / 零食 / 饮料
5. **常用食物列表**：我的收藏，快速添加
6. **饮食记录列表**：按餐次分组的今日记录，可删除/修改

**关键交互**：
- 搜索食物 → 搜索结果列表 → 点击食物 → 弹出份量选择 → 确认添加
- 点击分类 → 进入分类食物列表页
- 左滑记录项可删除
- 点击记录项可修改份量

---

##### 子页面：食物分类列表页

- 顶部：分类名称 + 食物数量
- 列表：每个食物显示名称、每100g热量、小图
- 点击食物 → 食物详情页

##### 子页面：食物详情页

- 食物大图（占位）+ 名称
- 每100g热量
- 营养成分表：蛋白质、碳水、脂肪、膳食纤维等
- 推荐克重提示
- 底部按钮：加入今日饮食（选择餐次+份量）

---

#### 页面3：断食（Fasting）

**核心模块**：
1. **当前断食状态大卡片**：
   - 状态（断食中/进食中）
   - 已断食时长 / 剩余时长
   - 大型圆形进度环
   - 开始/结束按钮
2. **断食模式选择**：
   - 16:8、18:6、20:4、OMAD、自定义
   - 可视化时间轴展示进食窗口
3. **今日时间线**：
   - 今天的断食/进食记录时间轴
4. **历史记录**：最近7天断食情况

**关键交互**：
- 切换断食模式 → 弹出确认框 → 确认后生效
- 点击开始断食 → 立即开始计时
- 自定义模式可设置进食开始时间和时长

---

#### 页面4：体重（Weight）

**核心模块**：
1. **当前体重卡片**：
   - 今日体重（大字）
   - 相比上次变化（↑↓ + 数字）
   - BMI 值 + 状态（偏瘦/正常/超重/肥胖）
2. **目标进度**：
   - 初始体重 → 当前体重 → 目标体重
   - 进度条 + 已减重百分比
   - 预计达标时间（P2）
3. **体重趋势折线图**：
   - 时间范围切换：7天 / 30天 / 全部
   - 折线图，每个点可点击看详情
4. **体重记录列表**：按日期倒序排列
5. **底部浮动按钮**：+ 记录体重

**关键交互**：
- 点击浮动按钮 → 弹出记录体重弹窗（输入体重 + 日期选择）
- 点击折线图数据点 → 显示当天体重详情
- 左滑记录可删除
- 点击记录可编辑

---

#### 页面5：我的（Profile）

**核心模块**：
1. **个人信息卡片**：
   - 头像 + 昵称
   - 身高、性别、年龄
   - 点击进入编辑页
2. **功能列表**：
   - 🏋️ 减肥目标设置
   - 📅 打卡日历
   - 📖 轻断食食谱（P1）
   - 💧 喝水记录（P2）
   - 🏃 运动记录（P2）
   - 🎨 主题设置（深色/浅色/跟随系统）
   - 📏 单位设置（公斤/斤、厘米/英尺）
   - 📤 数据导出（CSV）
   - 🔔 提醒设置
   - ℹ️ 关于
3. **统计数据**：累计减重、累计打卡天数、最长连续打卡

**关键交互**：
- 点击列表项进入对应页面
- 主题切换实时生效
- 数据导出生成文件后可分享/保存

---

##### 子页面：打卡日历

- 月视图日历
- 打卡的日期标圆点/色块
- 显示当月打卡天数
- 可切换月份

## 6. 技术选型建议

### 6.1 跨端框架选择

**推荐：React Native + Expo**

**理由**：
1. **开发效率高**：JavaScript/TypeScript 语言，一套代码跑 iOS + Android
2. **生态成熟**：社区活跃，第三方库丰富（图表、本地存储、通知等都有现成方案）
3. **Expo 加持**：不用配复杂的原生开发环境，快速启动，Over-the-Air 更新方便
4. **性能足够**：对于这种工具类App，性能完全够用
5. **方便调试**：Expo Go 可以直接在真机上预览

**备选方案对比**：

| 方案 | 优点 | 缺点 |
|------|------|------|
| **React Native (推荐)** | 生态好、上手快、人才多 | 原生交互略逊于Flutter |
| **Flutter** | 性能好、UI一致 | 要学Dart语言、生态不如RN |
| **uni-app** | 国内生态好、支持小程序 | 偏H5体验、复杂交互吃力 |
| **原生开发** | 体验最好 | 成本高、要维护两套代码 |

### 6.2 数据存储方案

**本地存储优先**（当前阶段）：

| 存储类型 | 方案 | 用途 |
|----------|------|------|
| **结构化数据** | SQLite（react-native-sqlite-storage 或 expo-sqlite） | 食物库、体重记录、饮食记录等 |
| **键值对/配置** | AsyncStorage 或 MMKV | 用户设置、用户信息、目标等 |
| **文件存储** | 文件系统 API（expo-file-system） | 导出的CSV、头像图片等 |

**为什么选 SQLite**：
- 食物库数据量大，需要高效查询和搜索
- 支持复杂查询（比如按分类、按热量范围筛选）
- 数据一致性好，支持事务
- 方便数据导入导出

**未来扩展**：如果后续需要云端同步，可以加一个后端服务（Node.js + MongoDB/PostgreSQL），做本地+云端双向同步。

### 6.3 图片/文件处理

| 类型 | 方案 |
|------|------|
| **App内置图片** | 打包在App资源里（食物图片、食谱图片、图标等） |
| **用户头像** | 存本地文件系统，路径存数据库 |
| **导出文件** | CSV生成后存本地，调用系统分享功能 |
| **占位图** | 先用纯色+图标，后续可以换成真实食物图 |

| **数据隐私** | 所有数据只存在用户手机本地，不上传服务器（除非用户主动备份） |
| **性能流畅** | 列表滚动60fps，页面切换无明显卡顿 |
| **包体积** | 安装包 < 50MB（iOS）/ < 30MB（Android） |
| **电量消耗** | 后台断食计时不能太耗电，用本地通知而非后台运行 |
| **兼容性** | iOS 13+ / Android 8.0+ |
| **数据安全** | 可选密码锁 / 生物识别（Face ID/指纹）进入App |
| **可恢复** | 支持本地备份和恢复，换手机可以迁移数据 |

## 8. 上线渠道

### 8.1 iOS

| 渠道 | 说明 |
|------|------|
| **App Store** | 正式上架，需要 Apple Developer 账号（99美元/年） |
| **TestFlight** | 内测方式，最多10000个测试用户，方便给朋友试用 |

### 8.2 Android

| 渠道 | 说明 |
|------|------|
| **Google Play** | 海外官方渠道，需要开发者账号（25美元一次性） |
| **国内应用市场** | 小米、华为、OPPO、vivo、应用宝等，每个都要单独上架审核 |
| **蒲公英 / Fir.im** | 内测分发，方便给朋友安装测试 |
| **直接发APK** | 最省事，直接把安装包发给朋友 |

### 8.3 内测阶段建议

1. **Android**：直接打包 APK 发给朋友，最方便
2. **iOS**：用 TestFlight，或者如果朋友不多的话用企业签名
3. **先做Android版本**：上架和测试都更灵活，等验证OK了再上iOS
